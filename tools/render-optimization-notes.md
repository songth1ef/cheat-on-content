# cheat-render 生成优化文档

> text2video.py 的画面/音频优化决策与待办。用户校准过的硬约束记这里，避免回退踩坑。

## 字体（可商用）⚠️ 重要

- **系统字体不可商用嵌入**：macOS 的 Hiragino Sans GB、Windows 微软雅黑都是随系统授权，**不能用于商业内容分发**。早期版本用过它们 → 有版权风险。
- **现用 = Noto Sans SC（思源黑体，SIL OFL 协议，免费商用、可嵌入分发）**，bundled 在 `tools/fonts/NotoSansSC.ttf`（可变字体，`wght 700/400` 切 Bold/Regular），跨平台一致。系统无 bundled 字体时才回退系统字体。
- 待办：17MB 可考虑用 `pyftsubset` 子集化到常用汉字（~3-4MB）省 git；但子集会限制可渲染字符，权衡后再做。
- 其它可商用备选（要换风格时）：阿里巴巴普惠体、HarmonyOS Sans、站酷系列（展示字重，适合标题）。

## 🎨 白色极简风格定稿（用户 2026-06-14，统一风格硬约束）

> 用户反复强调「极简主义 + 统一风格」。以下是定稿，改任何一条前先确认不破坏统一感。

- **主色 = 纯白，不要黄色**。大字幕、进度条、流程图箭头、昵称、进度数字全部白色。原黄色(255,212,0)已弃。
- **顶部描述行去掉**。不要 "AI 文章 · 看完不费劲" 这类 kicker。顶部只保留右上角头像 + 昵称「歌贼王」。
- **左下角进度数字 `i/n` 用白色 + 黑描边**，要清晰可见（原来灰色 120,130,145 几乎看不见）。
- **进度条 = 消耗式（draining）**：初始一整条纯白，播过的部分变透明，白条从满逐渐缩短到全部消失（不是从空填满）。无底槽。实现：白条 = `[fillw, W-90]`，`fillw` 随 t 右移。
- **流程图卡片 = 半透明白磨砂，不要纯黑底**。用 `img.alpha_composite` 真混合（白 alpha~212），照片微微透出来；**框内文字深灰(34,38,45)无描边**（白底够清晰，不要黑毛边）；连接箭头白色。
- **所有文字一律不要黑描边**（用户 2026-06-14 强调，很重要）：描边=黑毛边，难看。改用**柔和投影**（pop 大字幕用模糊暗影 GaussianBlur(6)；其余文字用偏移 3px 的半透明暗色 drop shadow）——不是描边，字边干净又能在亮背景上看清。
- **流程图磨砂白卡再浅**：`panel_a≈160`（原 212），照片透出更多、更轻盈。
- 头像右上角圆形 + 白圈；昵称「歌贼王」白字。

- **每条视频必须有结尾 CTA（引导互动），且每条不同**（用户 2026-06-14）：不要千篇一律"点赞关注"，要贴内容、口语、像真人随口问一句（如"你最想让 AI 替你干哪件事？评论区告诉我"）。在脚本最后一段加。
- **底部口播字幕最多 4 行 + 底部锚定 + 绝不重叠上方内容区**（用户 2026-06-14）：滑动窗口只显示最近几行（`oshown[-4:]`），底部锚 1648，满 4 行 top≈1380 > 内容区 ~1320。

## 画面（用户 2026-06-13/14 校准）

- **大字幕整行「弹出」（pop）**，不是逐字打字——打字太慢、初始露字太少。`pop_line` 缩放 0.72→1 + 淡入；多行依次整条弹。底部口播字幕才保留打字跟读。
- **背景 = 用户自拍照片**（`tools/backgrounds/<0-6>-<day>.jpg`，周一到周日固定循环；用户在该目录直接维护，不走桌面中转）。版权自有、可公开。
- **背景不加暗罩**（满亮度显示，用户要看清风景）：仅极轻模糊(2)+极轻提饱和(1.06)；**可读性全靠文字黑描边**（stroke），不靠压暗。
- **右上角头像 + 昵称「歌贼王」**：`tools/brand/avatar.jpg`（取自 blog 首页头像），圆形+白圈渲到每帧。
- **所有字体 ×1.2**（`FONT_SCALE=1.2`）。
- **流程图配图** `[图示] flow: A -> B -> C`：深色面板+accent 边+白字，逐个弹出。

## 文案忠实度（⚠️ 严重教训，用户 2026-06-13 指出）

- **报告/汇报类文案必须忠于事实，通俗≠抽象掉**。曾把「小程序码+条形码」泛化成「图片」，被批"文案飘逸"。专有名词、具体物料、数字、结论必须原样保留；只把纯技术实现翻成人话。
- 博客爆款可狠砍抽象，**报告绝不可以**——两种场景忠实度要求相反。
